While you are on the login page, try entering a wrong password. Because the security configuration uses .failureUrl("/auth/login?error=true"), Spring will redirect back to /auth/login?error=true, and the login page will show a friendly error message.

Together, the custom login page, the login processing URL, and the failure URL give you a clear, step-by-step view of how Spring Security handles both successful and failed authentication attempts before you end up on protected pages like this one.
